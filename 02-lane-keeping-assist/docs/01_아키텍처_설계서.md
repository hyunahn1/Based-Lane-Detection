## 1. 시스템 개요

### 1.1 목적

Module 01 (Lane Detection)의 출력을 기반으로 **차선 유지 보조 시스템(LKAS)**을 구현합니다. 차량이 차선을 벗어나지 않도록 실시간으로 모니터링하고, 필요 시 경고 및 자동 조향 개입을 수행합니다.

### 1.2 범위

**In Scope:**
- ✅ 차선 중심 추적 (Lane Center Tracking)
- ✅ 차선 이탈 감지 (Lane Departure Detection)
- ✅ 시각/청각 경고 시스템 (Warning System)
- ✅ PID 기반 조향 제어 (Steering Control)
- ✅ 안전 개입 메커니즘 (Safety Intervention)

**Out of Scope:**
- ❌ 차선 변경 보조 (Lane Change Assist)
- ❌ 충돌 회피 (Collision Avoidance) - Module 03에서 다룸
- ❌ 적응형 크루즈 컨트롤 (ACC)

### 1.3 시스템 요구사항 (RC 트랙 환경 기준)

| 카테고리 | 요구사항 | 목표 값 | 비고 |
|---------|---------|---------|------|
| **실시간성** | 제어 루프 주기 | < 50ms (20 Hz) | Python 구현 고려 |
| **정확도** | 차선 중심 추정 오차 | < 5cm | RC 트랙 스케일 (폭 35cm) |
| **응답성** | 차선 이탈 감지 시간 | < 100ms | 2-3 프레임 이내 |
| **안정성** | 조향 각도 변화율 | < 100°/s | RC 서보 응답 특성 |
| **안전성** | False Warning Rate | < 5 per 100 laps | RC 트랙 주행 기준 |

**⚠️ 중요 제약사항:**
- **환경**: 실내 RC 트랙 전용 (트랙 폭 30-40cm)
- **차량**: PiRacer (wheelbase ~0.25m)
- **카메라**: Pi Camera V2 (640×480)
- **정확도**: 카메라 캘리브레이션 없이는 ±30% 오차 가능

## 4. 핵심 컴포넌트

### 4.1 Lane Tracker (차선 추적기)

**책임 (Responsibility):**
- Module 01의 polyline을 분석하여 차선 중심선 추출
- 차량의 현재 위치(횡방향 오프셋) 계산
- 차선 곡률(curvature) 추정

**입력:**
```python
{
    "lane_polyline": List[Tuple[float, float]],  # (x, y) 좌표
    "vehicle_position": Tuple[float, float],      # 차량 중심 위치
    "image_shape": Tuple[int, int]                # (height, width)
}
```

**출력:**
```python
{
    "lane_center": List[Tuple[float, float]],    # 중심선
    "lateral_offset": float,                      # 횡방향 오프셋 (m)
    "heading_error": float,                       # 헤딩 오차 (degree)
    "curvature": float,                           # 곡률 (1/m)
    "is_valid": bool                              # 추적 유효성
}
```

**알고리즘:**
1. Polyline 스무딩 (Moving Average, Savitzky-Golay Filter)
2. 차량 좌표계로 변환 (Coordinate Transformation)
3. 최근접점 탐색 (Nearest Point on Curve)
4. 횡방향 거리 계산 (Cross-Track Error)
5. 헤딩 오차 계산 (Heading Error)

### 4.2 Departure Detector (이탈 감지기)

**책임:**
- 차선 이탈 여부 실시간 판단
- 이탈 위험도 레벨 계산
- Time To Crossing (TTC) 추정

**입력:**
```python
{
    "lateral_offset": float,
    "heading_error": float,
    "vehicle_speed": float,
    "lane_width": float
}
```

**출력:**
```python
{
    "is_departing": bool,
    "risk_level": int,  # 0-5 (0=safe, 5=critical)
    "time_to_crossing": float,  # seconds
    "departure_side": str  # "left", "right", "none"
}
```

**임계값 설정 (RC 트랙 기준 - 트랙 폭 35cm):**
| 레벨 | 횡방향 오프셋 | 헤딩 추정값 | 동작 |
|------|--------------|------------|------|
| 0 (Safe) | < 0.05m (5cm) | < 5° | 없음 |
| 1 (Normal) | 0.05-0.08m | 5-10° | 모니터링 |
| 2 (Caution) | 0.08-0.12m | 10-15° | 시각 경고 |
| 3 (Warning) | 0.12-0.15m | 15-20° | 청각 경고 |
| 4 (Critical) | 0.15-0.18m | 20-30° | 개입 준비 |
| 5 (Emergency) | > 0.18m | > 30° | 즉시 개입 |

**Note:**
- RC 트랙 폭이 35cm이므로 17.5cm가 중심선
- 18cm 이탈 = 트랙 경계 도달 (위험)
- Heading은 IMU 없이 polyline 변화율로 근사

### 4.3 Warning System (경고 시스템)

**책임:**
- 위험도에 따른 다단계 경고
- 운전자 주의 환기
- 경고 히스토리 관리

**경고 타입:**

1. **시각 경고 (Visual Warning)**
   ```python
   - Level 2: 노란색 차선 표시
   - Level 3: 주황색 깜빡임
   - Level 4: 빨간색 깜빡임 + 텍스트 "LANE DEPARTURE!"
   - Level 5: 전체 화면 빨간색 깜빡임
   ```

2. **청각 경고 (Auditory Warning)**
   ```python
   - Level 3: 부드러운 비프음 (1회)
   - Level 4: 반복 비프음 (2회)
   - Level 5: 긴급 경보음 (연속)
   ```

3. **햅틱 경고 (Haptic Warning)** - 향후 확장
   ```python
   - 조향 휠 진동
   - 차등 진동 (이탈 방향 표시)
   ```

### 4.4 Steering Controller (조향 제어기)

**책임:**
- PID 제어로 조향각 계산
- 부드러운 개입 (Smooth Intervention)
- 안전 제약 조건 만족

**PID 제어기 설계 (RC카 특화):**

```
Control Signal = Kp * error + Ki * ∫error dt + Kd * d(error)/dt + FF

where:
  error = lateral_offset + K_heading * heading_estimate
  FF = feedforward term = arctan(wheelbase * curvature)
  
Parameters (초기값 - RC카 튜닝 필요):
  Kp = 2.0  (비례 게인, RC카는 더 강한 제어 필요)
  Ki = 0.2  (적분 게인)
  Kd = 0.5  (미분 게인)
  K_heading = 0.2  (헤딩 가중치, heading 부정확하므로 낮게)
  
RC Car Specific:
  wheelbase = 0.25m (PiRacer)
  track_width = 0.35m (실내 트랙)
```

**안전 제약 (RC 서보 특성):**
- 최대 조향각: ±45° (RC 서보 범위)
- 최대 조향 속도: 100°/s (RC 서보는 빠름)
- Saturation 및 Anti-windup 적용

**⚠️ 주의:**
- 초기 파라미터는 추정값이며 실제 튜닝 필요
- PiRacer 하드웨어 특성에 따라 조정
- 실험적 튜닝 프로토콜 필요

**입력:**
```python
{
    "lateral_offset": float,
    "heading_error": float,
    "curvature": float,
    "vehicle_speed": float,
    "dt": float  # 시간 간격
}
```

**출력:**
```python
{
    "steering_angle": float,  # degree
    "throttle_adjustment": float,  # -1.0 ~ 1.0
    "confidence": float  # 0.0 ~ 1.0
}
```

## 6. 인터페이스 설계

### 6.1 External Interface (Module 01 연동)

**Module 01 실제 출력:**
```python
# Module 01 PostProcessor는 mask만 반환
{
    "lane_mask": np.ndarray,  # (H, W) binary mask {0, 1}
    "confidence": float       # 모델 신뢰도 (예: 평균 probability)
}
```

**Module 02 처리 전략:**

Module 02는 mask를 입력으로 받아 자체적으로 polyline을 추출합니다. 이는 **MSA 원칙**을 따릅니다:
- ✅ **독립성**: Module 02가 자신의 요구에 맞게 전처리
- ✅ **유연성**: 다른 lane detection 모듈과도 호환 가능
- ✅ **확장성**: Polyline 추출 알고리즘을 자유롭게 변경

**Module 02 Input Interface:**
```python
def process_frame(
    self,
    lane_detection: Dict,  # Module 01 출력
    vehicle_state: Dict
) -> Dict:
    """
    Parameters:
        lane_detection:
            {
                "lane_mask": np.ndarray,  # (H, W) binary mask
                "confidence": float       # 0.0 ~ 1.0
            }
        vehicle_state:
            {
                "speed": float,     # m/s (RC car speed)
                "timestamp": float  # optional, Unix timestamp
            }
    """
    pass
```

**Output Interface:**
```python
class LKASOutput:
    """Module 02 출력 형식"""
    steering_angle: float           # 조향각 (degree)
    throttle_adjustment: float      # 스로틀 조정 (-1.0 ~ 1.0)
    warning_level: int              # 경고 레벨 (0-5)
    is_intervening: bool            # 개입 여부
    lateral_offset: float           # 횡방향 오프셋 (m)
    heading_error: float            # 헤딩 오차 (degree)
    timestamp: float
```

### 6.2 Configuration Interface

**설정 파일: `config/lkas_params.yaml` (RC 트랙 최적화)**
```yaml
# RC Car Environment
environment:
  wheelbase: 0.25  # meters (PiRacer)
  track_width: 0.35  # meters (실내 RC 트랙)
  max_speed: 2.0  # m/s (약 7 km/h)
  camera_height: 0.15  # meters (지면으로부터)
  camera_angle: 15  # degrees (하향 각도)

# Pixel-Meter Conversion (경험적 근사)
calibration:
  use_perspective_correction: true
  track_width_pixels_bottom: 384  # 이미지 하단에서 트랙 폭 (픽셀)
  perspective_scale_top: 3.0  # 상단 픽셀 스케일 (먼 거리)
  
# Tracking Parameters
tracking:
  smoothing_window: 5
  min_confidence: 0.6  # RC 트랙은 높은 신뢰도 필요
  max_lateral_offset: 0.20  # meters (트랙 절반 이상 벗어나면 위험)

# Departure Detection (RC 트랙 스케일)
departure:
  risk_thresholds:
    level_2: 0.08  # 8cm (트랙 폭의 ~25%)
    level_3: 0.12  # 12cm
    level_4: 0.15  # 15cm
    level_5: 0.18  # 18cm (거의 트랙 경계)
  heading_thresholds:
    level_2: 10.0   # degrees
    level_3: 20.0
    level_4: 30.0
    level_5: 40.0

# PID Controller (RC Car Tuned - 초기 추정)
controller:
  kp: 2.0  # 강한 비례 제어 (작은 스케일)
  ki: 0.2
  kd: 0.5  # 강한 미분 (빠른 변화 대응)
  k_heading: 0.2  # heading 근사값이므로 낮은 가중치
  max_steering_angle: 45.0  # degrees (RC 서보 범위)
  max_steering_rate: 100.0  # deg/s (RC 서보는 빠름)
  windup_limit: 5.0  # 작은 스케일에 맞춤

# Warning System
warning:
  enable_visual: true
  enable_audio: true
  enable_haptic: false  # PiRacer에 햅틱 없음

# Safety Limits (RC Car)
safety:
  max_intervention_duration: 3.0  # seconds
  min_speed_for_lkas: 0.5  # m/s (매우 낮은 속도는 제외)
  max_speed_for_lkas: 2.0  # m/s (고속에서 불안정)
  emergency_brake_threshold: 0.18  # meters (트랙 경계)

# Logging
logging:
  level: INFO
  save_telemetry: true
  telemetry_dir: logs/
```

## 8. 성능 목표

### 8.1 정량적 목표 (KPI) - RC 트랙 환경

| 메트릭 | 목표 | 측정 방법 | 우선순위 |
|--------|------|-----------|---------|
| **처리 지연시간** | < 30ms | Frame-to-command time | P0 (필수) |
| **차선 중심 추정 MAE** | < 5cm | 실측 데이터 대비 | P0 (필수) |
| **이탈 감지 Precision** | > 85% | TP / (TP + FP) | P0 (필수) |
| **이탈 감지 Recall** | > 90% | TP / (TP + FN) | P0 (필수) |
| **False Warning Rate** | < 5 per 100 laps | RC 트랙 주행 | P1 (권장) |
| **조향 안정성** | Jerk < 50°/s² | 조향각 2차 미분 | P1 (권장) |
| **CPU 사용률** | < 40% | 단일 코어 (RPi) | P2 (선택) |
| **메모리 사용량** | < 100MB | RSS | P2 (선택) |

**성능 목표 설정 근거:**

| 항목 | 이유 |
|------|------|
| **30ms** | Python 구현 + NumPy 연산 고려, 여유 확보 |
| **5cm** | RC 트랙 폭(35cm)의 ~14%, 측정 불확실성 ±3cm 고려 |
| **85% Precision** | 초기 구현, 튜닝 후 개선 목표 |
| **90% Recall** | 안전 우선, 놓치는 것보다 과검출 선호 |
| **5/100 laps** | RC 트랙 1바퀴 ≈ 20m, 2000m = 2km 기준 |

### 8.2 정확도 제약사항 명시

**⚠️ 측정 불확실성:**
- **픽셀-미터 변환 오차**: ±30% (카메라 캘리브레이션 없음)
- **Heading 추정 오차**: ±10° (IMU 없이 polyline으로 근사)
- **곡률 추정 오차**: ±50% (3점 근사 + 변환 오차)

**권장 개선사항:**
1. **Camera Calibration**: 체커보드 패턴으로 내부/외부 파라미터 추정
2. **IPM (Inverse Perspective Mapping)**: 원근 왜곡 제거
3. **IMU 센서 추가**: 정확한 차량 heading 측정
4. **Ground Truth 수집**: 실제 측정값으로 보정 계수 튜닝

## 9. 보안 및 안전

### 9.1 안전 메커니즘

**Fail-Safe 설계:**
```python
1. Input Validation
   - 차선 검출 신뢰도 체크 (confidence > threshold)
   - 센서 데이터 일관성 검증
   - 이상치 필터링

2. Watchdog Timer
   - 30ms 내 응답 없으면 안전 모드 진입
   - 제어 권한 운전자에게 즉시 반환

3. Graceful Degradation
   - Module 01 실패 시: 경고만 유지, 개입 중단
   - 센서 이상 시: 즉시 개입 중단, 경고 발생

4. Emergency Override
   - 운전자 조향 입력 감지 시 즉시 제어 반환
   - 브레이크 입력 시 시스템 일시 정지
```
