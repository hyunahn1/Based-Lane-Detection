# 문서 수정 이력

## 2026-01-29: v2.0.1 - 팩트체크 반영 (주요 수정)

### 배경
GPT Codex의 팩트체크 결과, 다음 문제점 발견:
1. 문서 간 수치 불일치
2. 과장된 목표 설정 (근거 부족)
3. 기술적 오류 (MMSeg API 오용)
4. 검증 로직 오류

### 주요 수정 사항

#### 1. 수치 통일 ✅

| 항목 | 이전 (v2.0) | 수정 (v2.0.1) |
|------|-------------|---------------|
| 증강 수량 | "1,000~1,500개" | **"695개"** (Train 139×5) |
| 손실 함수 | 4가지 복합 | **CrossEntropy + Dice** (단순화) |
| 배치 사이즈 | "64 가능" | **"16부터 시작, 실측 필요"** |
| IoU 목표 | "0.85+" (확정적) | **"0.75-0.80"** (현실적) |
| 개발 기간 | "3-5일" | **"2주"** (현실적) |
| 스케줄러 | WarmRestarts | **CosineAnnealing** (단순) |

#### 2. 기술적 오류 수정 ✅

**MMSeg API 수정**:
```python
# ❌ 잘못된 방식
output = model(x)
pred = output['pred_sem_seg'].data

# ✅ 올바른 방식
result = inference_model(model, image)
pred = result.pred_sem_seg.data
```

**메트릭 계산 수정**:
- GT 값 범위: 0/1 또는 0/255 모두 허용
- Pred 값 범위: 클래스 인덱스 (0 or 1)
- 0으로 나누기 방지 추가

**검증 임계값 수정**:
- 최소 기준: IoU ≥ 0.70 (0.80에서 하향)
- 3단계 평가: 최소/현실적/희망

#### 3. 과장 표현 수정 ✅

**이전 (과장)**:
- "3-5일 안에 IoU 0.85+ 달성"
- "배치 64 가능"
- "확정적" 톤

**수정 (현실적)**:
- "2주 안에 IoU 0.75-0.80 목표 (실험 기반 업데이트)"
- "배치 16부터 시작, 실측 필요"
- "⚠️ 추정치" 경고 명시

#### 4. 문서 간 일관성 확보 ✅

모든 문서에서:
- 증강 수량: 695개로 통일
- IoU 목표: 0.75-0.80으로 통일
- 배치: 16부터 시작으로 통일
- 기간: 2주로 통일
